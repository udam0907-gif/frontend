# REPORT_TEMPLATE.md — 작업 보고 양식

> 모든 작업 보고는 이 양식을 따른다.
> 양식을 벗어나거나 항목을 생략하면 보고가 무효 처리된다.

---

## 📋 표준 보고 양식

### ✅ 통과 보고 (작업 성공 시)

```markdown
## 작업 보고

### 1. 무엇을 했나
- 작업 지시: [원본 사용자 지시 한 줄 요약]
- 수정 파일:
  - `[파일경로 1]` — [변경 요약]
  - `[파일경로 2]` — [변경 요약]
- 변경 라인 수: +XX / -XX

### 2. 검증 증거 ⭐
- **컨테이너 상태**:
  ```
  [docker ps 출력 일부]
  ```
- **API 호출 결과**:
  ```bash
  $ curl -X POST http://localhost:8000/...
  HTTP 200
  {응답 본문 일부}
  ```
- **로그 확인**: 새 에러 없음 (확인 완료)
- **회귀 테스트**: 기존 [엔드포인트 목록] 정상

### 3. 영향받은 다른 기능
- [없음 / 있음: 무엇]

### 4. 사용자가 직접 확인할 것
- [ ] [브라우저에서 확인할 화면 또는 동작]
- [ ] [기타 수동 확인 항목]

### 5. 다음 단계 제안
- [추후 개선 사항 또는 다음 작업]

### 6. git 상태
$ git log --oneline -3
[최근 커밋 3개]

권장 커밋 메시지: `[메시지 제안]`
```

### 2. 무엇이 안 됐나

### 3. 증거 (실패의 객관적 근거)
[에러 로그 또는 명령 출력]

### 4. 원인 분석
- 추정 원인: [가능성 높은 것부터]
- 확신도: [높음/중간/낮음]

### 5. 다음 시도 제안
1. [방법 1]
2. [방법 2]

### 6. 사용자 결정 필요 사항
- [ ] [선택지 A] vs [선택지 B] 중 어느 쪽?
- [ ] [추가 정보 필요]

### 7. 현재 상태
- 코드: [원복됨 / 변경 상태로 둠]

```

### 현재 상태
- 변경이 적용됨 / 작업 트리에만 있음 / 이미 commit 됨

### 제안 조치
- [원복 명령 또는 사용자 결정 요청]

```

## 📝 작성 규칙

### 필수
- 코드 블록은 실제 명령 출력을 그대로 넣는다 (요약/편집 금지)
- 파일 경로는 프로젝트 루트 기준 풀 경로
- 영어/한글 혼용 시 코드는 영어, 설명은 한글
- 추측은 명시 ("추정", "확신 낮음" 등 표시)

### 금지
- ❌ "잘 됐을 것입니다" 같은 추측 표현
- ❌ "거의 다 됐어요" 같은 모호한 진행률
- ❌ 검증 증거 없이 "테스트 통과" 주장
- ❌ 사용자가 모르는 변경 숨기기
- ❌ 보고를 보고서 양식 없이 산문으로 길게 늘어놓기

---

## 🎯 보고 길이 기준

| 작업 규모 | 보고 길이 |
|----------|---------|
| 단순 수정 (1~10줄) | 짧게: 1~2개 코드블록 + 핵심만 |
| 중간 작업 (10~100줄) | 양식 모든 섹션 |
| 큰 작업 (100줄+) | 양식 + 별도 변경 사항 요약 |

**원칙: 정보는 빠짐없이, 텍스트는 최소한.**

---

## 💬 사용자에게 질문이 필요할 때

작업 도중 사용자 결정이 필요하면 **즉시 멈추고** 다음 양식으로 질문:

```markdown
## ⏸️ 사용자 결정 필요

### 현재 상황
[어디까지 했는지]

### 막힌 지점
[무엇 때문에 멈췄는지]

### 선택지
A. [옵션 A] — 장점: ..., 단점: ...
B. [옵션 B] — 장점: ..., 단점: ...
C. [옵션 C] — ...

### 권장
[추천하는 옵션과 그 이유]

→ 답변 기다리는 중 (작업 일시 정지)
```

추측해서 진행하지 않는다. 답이 올 때까지 기다린다.

## ✅ 최종 보고 직전 셀프 체크

- [ ] 양식의 모든 섹션이 채워졌는가?
- [ ] 검증 증거가 실제 실행 결과인가? (추측 아닌가?)
- [ ] 보호 영역 침범 검사 완료했는가?
- [ ] 사용자가 직접 확인할 것을 명시했는가?
- [ ] 거짓이 들어갈 여지가 없는가?

5개 모두 YES일 때만 보고한다.
